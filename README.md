Course **INE5458 — Blockchain and Cryptocurrency Technologies** — UFSC 2026/1

A decentralized voting platform for the São José/SC participatory budget. Every stage of the cycle (voter registration, proposal submission, voting, tallying) is recorded on a public blockchain (Polygon PoS), removing the central trusted authority and enabling open auditing by any citizen.

## Running the project

Every command below is a plain `npm run <script>`, `npm test`, or `npx hardhat ...` call — they work identically on **Linux, macOS, and Windows** because the OS-specific parts (quoting, creating the `reports/` folder, etc.) are already handled inside the npm scripts, not left to your shell. The only two places where the shell itself matters are **chaining multiple commands** and **copying a file** — both are called out explicitly below with a command for each platform. Windows users can run any of this from **PowerShell** or **Git Bash**; both work, pick whichever you already have open.

### Prerequisites

- **Node.js 20 LTS.** Hardhat does not officially support Node 22+; newer versions work but print a warning.
  - **Linux / macOS**, with [nvm](https://github.com/nvm-sh/nvm):

    ```bash
    nvm install 20
    nvm use 20
    ```

  - **Windows**, with [nvm-windows](https://github.com/coreybutler/nvm-windows) (same commands, PowerShell or cmd):

    ```powershell
    nvm install 20
    nvm use 20
    ```

  - No `nvm`? Install Node 20 LTS directly from [nodejs.org](https://nodejs.org/) on either platform.
- **Git** and an editor.

### Chaining commands: a note for Windows PowerShell

Windows PowerShell 5.1 (the version that ships with Windows) does **not** support the `&&` chain operator used throughout Linux/macOS documentation and examples online. Where this README shows `command-a && command-b`, use one of these on PowerShell instead:

```powershell
# Run command-a, then command-b only if command-a succeeded
command-a; if ($?) { command-b }

# Or just run them on two separate lines — simplest, works the same way
command-a
command-b
```

Git Bash on Windows supports `&&` natively, same as Linux/macOS, if you'd rather use that instead of PowerShell.

### 1. Install dependencies

```bash
npm install
```

Identical on Windows (PowerShell or Git Bash) and Linux/macOS. No keys or environment variables are required at this point — installation is fully offline after the initial `npm install`.

**Shortcut:** `npm run verify` runs everything below in one go — lint, tests (with the gas report), coverage, structural metrics, and the end-to-end demo — and exits non-zero if anything fails.

```bash
# Linux / macOS / Git Bash
npm install && npm run verify
```

```powershell
# Windows PowerShell
npm install; if ($?) { npm run verify }
```

### 2. Run the test suite

```bash
npm test          # compiles + runs the full Hardhat test suite (43 tests) — also prints a per-function gas report
npm run coverage  # same suite under istanbul — 100% statements/branches/functions/lines
npm run lint      # solhint + eslint + markdownlint
```

One of those 43 is [`test/LargeScaleCycle.test.js`](test/LargeScaleCycle.test.js): it generates **500 independent wallets** (not reused signers), registers all of them, runs 6 proposals, casts 500 votes with a deliberately uneven split, and cross-checks the contract's tally against an independently tracked count in JavaScript — for every proposal, not just the winner. It finishes in a few seconds; push it further with `VOTASJ_SCALE_VOTERS=2000 npm test` (~16s, gas per vote stays flat either way).

`npm test` also prints a gas-cost table for every contract method (min/max/avg gas and % of the block gas limit for each deployment), written to `reports/gas-report.txt`. The same checks run on every push and pull request via GitHub Actions — see [.github/workflows/ci.yml](.github/workflows/ci.yml). `lint` and `test` run on a matrix of **Ubuntu and Windows** (the `test` job also across **Node 20 and 22**), so a Windows-only or Node-version-only regression fails the build instead of surfacing only on someone's laptop. The `coverage` job fails the build if any of statements/branches/functions/lines drops below 100% — coverage is a gate here, not just a number in a badge.

### 3. Run the end-to-end evidence demo

```bash
npm run demo
```

This is the fastest way to see the whole flow work and to get proof it worked. [`scripts/demo-cycle.js`](scripts/demo-cycle.js) deploys both contracts, registers three citizens, opens a cycle, submits three proposals, casts votes, closes the cycle, reads the tally back from the contract, and — just as importantly — tries five illegal actions (double registration, an unregistered address proposing, an unregistered address voting, double voting, closing early) and asserts every one of them reverts with the exact reason the tests document. It prints every transaction hash, block number, gas cost, and decoded event as it happens, and writes the same information to `reports/demo-cycle-report.md` (human-readable) and `reports/demo-cycle-report.json` (machine-readable) so the run can be inspected after the fact or attached as evidence.

This same command runs in CI on every push, **on both Ubuntu and Windows runners** (see the `demo` job in [.github/workflows/ci.yml](.github/workflows/ci.yml)), and each run's report is uploaded as a build artifact — anyone can check the Actions tab and download the evidence from any run without cloning the repo. A committed example is at [docs/pd3/evidence/sample-demo-report.md](docs/pd3/evidence/sample-demo-report.md).

### 4. Look at the structural metrics

```bash
npm run metrics
```

[`scripts/contract-metrics.js`](scripts/contract-metrics.js) reads the Solidity source and the compiled artifacts and reports, per contract: lines of code, function/event/modifier/state-variable counts, a heuristic cyclomatic complexity per function (methodology documented at the top of the script), and the deployed bytecode size against Ethereum's 24576-byte (EIP-170) contract-size limit. It writes `reports/contract-metrics.md` and `.json`. This is the structural counterpart to the behavioral evidence from `npm run demo` — proof the contracts are small and simple enough to audit, not just that they pass their own tests. Runs in CI on every push (`metrics` job), report uploaded as a build artifact.

### 5. Run a local blockchain and deploy against it

In one terminal, start a local JSON-RPC node with 20 pre-funded test accounts:

```bash
npm run node
```

The node prints the accounts and their private keys — these are **publicly known test keys**, never use them on any real network. Leave this terminal running.

In a second terminal, deploy the contracts to the local node:

```bash
npm run deploy:local
```

The deploy script prints the addresses of `VoterRegistry` and `ParticipatoryBudget`. Because the local node is deterministic, the first deployment always lands on the same addresses.

### 6. Interact with the contracts

From a third terminal, open an interactive console attached to the local node:

```bash
npx hardhat console --network localhost
```

Example of a full cycle inside the console — registration, opening a cycle, submitting a proposal, voting, and closing:

```javascript
const registry = await ethers.getContractAt("VoterRegistry", "<REGISTRY_ADDRESS>");
const budget   = await ethers.getContractAt("ParticipatoryBudget", "<BUDGET_ADDRESS>");
const [admin, alice, bob] = await ethers.getSigners();

await registry.registerVoter(alice.address, ethers.keccak256(ethers.toUtf8Bytes("govbr:alice")));
await registry.registerVoter(bob.address,   ethers.keccak256(ethers.toUtf8Bytes("govbr:bob")));

const now = (await ethers.provider.getBlock("latest")).timestamp;
await budget.openCycle(now + 1, now + 3600, ethers.parseEther("100"));

await budget.connect(alice).submitProposal("ipfs://QmExample");
await budget.connect(alice).vote(1);
await budget.connect(bob).vote(1);

// fast-forward past the cycle deadline
await network.provider.send("evm_increaseTime", [3600]);
await network.provider.send("evm_mine");

await budget.closeCycle();
(await budget.cycles(1)).winningProposalId;  // -> 1n
```

> REPL tip: do not start a line with `.` in the Node REPL — it is interpreted as a REPL command (`.help`, `.exit`). Keep method chains on a single line.

### 7. Deploy to Polygon Amoy (public testnet)

For any deployment to a real network you need two secrets:

- **RPC URL** of a Polygon Amoy node.
- **Private key** of a deployer account, funded with test MATIC.

Neither is checked into the repo. Create a `.env` file at the repository root (it is in `.gitignore`):

```bash
# Linux / macOS / Git Bash
cp .env.example .env
```

```powershell
# Windows PowerShell
Copy-Item .env.example .env
```

Then fill in the two variables:

| Variable | How to obtain |
| -------- | ------------- |
| `AMOY_RPC_URL` | Free public endpoint: `https://rpc-amoy.polygon.technology` — works without signup but is rate-limited and occasionally unreliable. For production use, get a dedicated endpoint at [Alchemy](https://www.alchemy.com/) or [Infura](https://www.infura.io/) — free tier, sign up with email, create an app targeting `Polygon Amoy`, copy the HTTPS URL. |
| `DEPLOYER_PRIVATE_KEY` | Create a fresh wallet in MetaMask, export the private key — this is the easiest path on any OS. **Use a throwaway account, not a wallet that ever touched real funds.** Alternatively generate a random one yourself: `openssl rand -hex 32` (Linux/macOS/Git Bash) or `-join ((1..32) \| ForEach-Object { '{0:x2}' -f (Get-Random -Max 256) })` (PowerShell). Then fund the corresponding address with test MATIC from the [Polygon Amoy faucet](https://faucet.polygon.technology/) — paste the wallet address, pick Amoy, request. You get ~0.2 test MATIC per request, enough for many deployments. |

After filling in `.env`:

```bash
npm run deploy:amoy
```

This sends two real transactions to Amoy and prints the deployed addresses. They can be inspected at [amoy.polygonscan.com](https://amoy.polygonscan.com/).

### Keys summary

| Context | Keys needed |
| ------- | ----------- |
| `npm test` / `npm run demo` | None — in-process network. |
| `npm run node` + `npm run deploy:local` | None — Hardhat generates 20 test accounts with publicly known keys. |
